# 现场真机演示说明

本功能要求能在现场用真实设备演示。推荐准备一台电脑作为发送/接收服务端，再准备一台手机或平板作为另一端设备。

## 演示前准备

1. 电脑和手机/平板连接同一个 Wi-Fi。
2. 电脑关闭 VPN 的全局/TUN 接管，避免局域网地址不可达。
3. 如果 Windows 防火墙弹出 Node.js 入站访问提醒，选择允许当前网络访问。
4. 准备一个小文件，例如图片、PDF 或文本文件，便于快速展示上传和下载。

## 启动服务

在电脑上运行：

```powershell
cd wireless-file-transfer
.\Start-Demo.ps1
```

终端会显示 PIN、`Local` 地址和一个或多个 `LAN` 地址。优先使用带 `recommended` 标记的 `LAN` 地址。

## 真机演示流程

1. 电脑浏览器打开 `http://127.0.0.1:5731`。
2. 手机/平板浏览器打开终端显示的 `LAN` 地址。
3. 电脑端上传一个文件，手机/平板端点击“刷新列表”后下载该文件。
4. 反过来，手机/平板端选择一个文件上传，电脑端刷新后下载。
5. 下载成功后，删除演示文件，说明文件列表和清理能力也可用。

## 判断是否真的跨设备

- 手机/平板地址栏访问的是电脑的局域网 IP，例如 `http://192.168.x.x:5731`、`http://10.x.x.x:5731` 或可达的 `http://172.x.x.x:5731`。
- 手机/平板不是通过数据线，也不是模拟器内部访问。
- 电脑端上传后，手机/平板端可以下载；手机/平板端上传后，电脑端也可以下载。

## 现场故障处理

- 打不开页面：确认两台设备在同一 Wi-Fi，并优先使用页面标记为“推荐真机”的地址。
- 仍打不开页面：检查 Windows 防火墙是否允许 Node.js 入站访问。
- 上传失败：换一个更小的文件先演示，确认网络连通后再传大文件。
- 页面能打开但列表不更新：点击“刷新列表”，或重新打开页面。
